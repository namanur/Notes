# Strategy Log Skill
**Purpose:** Capture architectural decisions, directional changes, and the "why" behind system pivots.
**When to use:** When shifting between learning phases, starting new projects, or re-architecting existing vault layers.
**Steps:**
1. Identify the project or system layer being modified.
2. Locate or create a `REBUILD_STRATEGY.md` or `REBUILD_LOG.md` (e.g., in `08_Work/ProjectName/`).
3. Summarize the current state and the core problem being solved.
4. Document the chosen architectural path and alternatives considered.
5. Record the technical rationale behind the decision.
**Output:** A strategic record of system evolution and decision logic.
