# Ubiquitous Language — Agent Architect Manual

This document defines the binding terminology for the manual. Do not deviate from these definitions.

## Core Concepts

- **Agent**: A software entity capable of perception, memory, reasoning, and action to achieve a goal.
- **Harness**: The minimum execution environment (code + configuration + tools) that allows an agent to function.
- **Perception**: The process of receiving and parsing input (text, files, system state).
- **Memory**: Persistent or short-term storage of information. Includes **In-context** (short-term) and **SQLite** (long-term).
- **Reasoning**: The logical processing of input and memory to determine the next action.
- **Action**: The execution of a tool or command to modify the environment or generate output.
- **Output**: The final result or communication generated by the agent.

## Technical Components

- **MCP (Model Context Protocol)**: A standard for connecting AI models to data sources and tools.
- **Tool**: A specific function or script an agent can call (e.g., a bash script or a Python function).
- **Resource**: Data provided to an agent via MCP (e.g., a file content or a database view).
- **Orchestrator**: An agent that coordinates other agents or manages a multi-step workflow.
- **Context Window**: The maximum amount of information an LLM can process in a single turn.

## Methodologies

- **DDD (Domain-Driven Design)**: Designing agents around specific business domains and language.
- **TDD (Test-Driven Development)**: Writing tests before implementing agent behavior to ensure consistency.
- **Grill Me**: An adversarial validation process where an output is challenged by an independent agent.
